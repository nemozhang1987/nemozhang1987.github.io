在mac pro中利用hexo在github建博客的痛苦过程

  • hexo,node.js,mac,git

经历了两天的折腾之后，终于把博客网站搞定了，着实不易啊，参考了多篇文章的说明，遇到了各种困难….
下面正式开篇。
PS，虽然hexo已经帮我自动生成了一篇文章。

基础环境准备

hexo是基于node.js的，同时我们需要使用github做为blog的web服务器（其实就是提供了子域名的公网访问，当然还有文件服务。Github Pages可以被认为是用户编写的、托管在github上的静态网页。），所以还需要git，以及ssh的公钥，目前mac系统中实际上已经集成了git，所以不需要安装，仅做配置即可。

环境

  1. Mac Book Pro 硬件
  2. Mac OS X EI Captian 10.11.5 操作系统
  3. git 软件Mac已自带
  4. brew包管理器 软件待安装
  5. Node.js 软件待安装
  6. Hexo 基于Node.js的博客页面软件
  7. github账号 没有这个没法在github上托管
  8. ssh 软件Mac已自带

github以及ssh key设置

如果你没有github账号，请先注册一个，地址到https://www.github.com

绑定Git & Github

创建博客仓库

注册帐号后，点➕来创建一个repository，该仓库必须以 “your_user_name.github.io” 的格式命名,如下图所视：

这样你就可以以[https://your_user_name.github.io] 来访问你的hexo博客了，当然现在还不行，我们还没上传文件，还需经历下一步才可以。

配置SSH密钥

配置Github的SSH密钥可以让本地git项目与远程的github建立联系，让我们在本地写了代码之后直接通过git操作就可以实现本地代码库与Github代码库同步。操作如下
ls ~/.ssh
如果没有该目录则进行下一步，如果有该目录，且目录下有内容，则需要在后续的步骤中调整对应的sshkey文件名。

创建ssh公钥对

直接回车，则将密钥按默认文件进行存储。此时也可以输入特定的文件名，比如/c/Users/you/.ssh/github_rsa
接着，根据提示，你需要输入密码和确认密码（说到这里，如果你很放心，其实可以不用密码，就是到输密码的地方，都直接回车，所以每次push就只管回车就行了。所谓的最安全的密码，就是没有密码 哈哈）。相关提示如下：
Enter passphrase (empty for no passphrase): [Type a passphrase]
Enter same passphrase again: [Type passphrase again]
输入完成之后，屏幕会显示如下信息：

Your identification has been saved in /Users/you/.ssh/id_rsa.
Your public key has been saved in /Users/you/.ssh/id_rsa.pub.
The key fingerprint is:
01:0f:f4:3b:ca:85:d6:17:a1:7d:f0:68:9d:f0:a2:db your_email@example.com

将sshkey内容拷贝到剪贴板

后面我们将密钥内容设置到github账户中，以授权访问
在github已登陆页面，选择自己的头像，并下拉选择settings

点击New SSH key

将刚才复制进来的sshkey内容复制到文本框中，并点add sshkey即可
测试SSH连接

    进入到hexo博客文件目录，删除 _.deploygit 目录即可。
